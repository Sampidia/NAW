üéØ AYO GAME - IMAGE REPLACEMENT GUIDE
=====================================

This implementation provides an easy system for replacing the wooden board and pits with your own PNG images.

üìÅ IMAGE REPLACEMENT FILES:
===========================

1. BOARD BACKGROUND:
   - Current: app/src/main/res/drawable/board_background.xml
   - Replace with: app/src/main/res/drawable/board_background.png
   - Purpose: The main wooden board background
   - Recommended size: 800x400px (will be scaled)

2. NORMAL PIT:
   - Current: app/src/main/res/drawable/pit_normal.xml
   - Replace with: app/src/main/res/drawable/pit_normal.png
   - Purpose: Regular pit appearance
   - Recommended size: 60x60px

3. ACTIVE PIT:
   - Current: app/src/main/res/drawable/pit_active.xml
   - Replace with: app/src/main/res/drawable/pit_active.png
   - Purpose: Highlighted pit when selectable
   - Recommended size: 60x60px

üîß HOW TO REPLACE IMAGES:
==========================

1. Create your PNG images with transparent backgrounds
2. Save them in: app/src/main/res/drawable/
3. Use EXACTLY these names:
   - board_background.png (for the wooden board)
   - pit_normal.png (for regular pits)
   - pit_active.png (for highlighted pits)

4. Build and run - your custom images will automatically load!

üéÆ FULLSCREEN MODE:
===================

The game now runs in fullscreen mode for better immersion:
- Status bar is hidden during gameplay
- Navigation bar is hidden
- Screen stays awake during gameplay
- Swipe from edges to temporarily show system UI

üì± SUPPORTED ANDROID VERSIONS:
=============================

- Android 11+ (API 30+): Uses modern WindowInsetsController
- Older versions: Uses legacy fullscreen flags
- Automatic fallback handling

üîÑ BACKWARD COMPATIBILITY:
==========================

If your custom PNG files are not found, the system automatically falls back to the default wooden appearance, so your game will always work.

üé® CUSTOMIZATION TIPS:
======================

1. For best results, use high-quality PNG images with transparent backgrounds
2. Maintain 60x60px dimensions for pits for consistent layout
3. Use warm brown tones (#8B4513, #A0522D #8B4513) for authentic wooden appearance
4. Consider adding subtle shadows for depth

üöÄ ADVANCED CUSTOMIZATION:
==========================

For even more control, you can modify:
- MainActivity.kt: Fullscreen implementation and image loading
- ImageManager.kt: Custom image loading logic
- activity_main_images.xml: Layout structure

The system is designed to be flexible and easy to customize while maintaining reliability!

## Image Size Recommendations

### __1. `game_page_background.png`__ - Main Game Page

__Recommended: 1920x1080px or higher__

- __Why this size:__ Full-screen background, should cover entire game area
- __Aspect ratio:__ 16:9 (landscape orientation)
- __Target devices:__ From 720p to 4K displays
- __File size:__ Keep under 500KB for performance

__Alternative sizes:__

- __Minimum:__ 1280x720px (HD)
- __Standard:__ 1920x1080px (Full HD)
- __High-end:__ 2560x1440px (2K)

### __2. `game_over_background.png`__ - Game Over Dialog

__Recommended: 800x600px__

- __Why this size:__ Dialog is approximately 300dp wide, needs comfortable padding
- __Aspect ratio:__ 4:3 (dialog-friendly ratio)
- __Usage:__ Centered dialog background with text overlay
- __File size:__ Keep under 200KB

__Alternative sizes:__

- __Compact:__ 600x450px
- __Standard:__ 800x600px
- __Large:__ 1000x750px

### __3. `game_status_background.png`__ - Floating Status Card

__Recommended: 400x200px__

- __Why this size:__ Small floating card (200dp minWidth) for status messages
- __Aspect ratio:__ 2:1 (horizontal card layout)
- __Usage:__ Player turn indicator with minimal text
- __File size:__ Keep under 100KB

__Alternative sizes:__

- __Compact:__ 300x150px
- __Standard:__ 400x200px
- __Wide:__ 500x250px

## üéØ Design Considerations

### __For All Background Images:__

- __Format:__ PNG with transparency support recommended
- __DPI:__ 72-96 DPI (screen resolution)
- __Colors:__ Consider text readability (white text is used)
- __Compression:__ Optimize for web/mobile (80-90% quality)

### __Performance Tips:__

- __File size limits:__ Keep each under 500KB to avoid memory issues
- __Memory usage:__ Android loads full-size images, so smaller is better
- __Caching:__ System will cache optimized versions automatically

### __Responsive Design:__

- __Scaling:__ Images will be scaled to fit their containers
- __Aspect ratio:__ Maintain consistent ratios for best results
- __Edge handling:__ Consider how images look when cropped or stretched

## üì± Target Device Coverage

__These sizes will work well for:__

- __Phones:__ 4.7" to 6.5" screens (720p to 1080p)
- __Tablets:__ 7" to 10" screens (1080p to 1440p)
- __Orientation:__ Both portrait and landscape modes

__For the game page background especially:__ Consider creating multiple versions for different screen densities if you want pixel-perfect results.

## `TooltipCompat`

`TooltipCompat` is Android's built-in solution for showing contextual information on touch/long-press. It's designed specifically for mobile "hover" effects and provides excellent UX.

### Key Benefits:

- __Native Android UX__: Uses platform-standard visual styling and behavior
- __Accessibility Support__: Built-in screen reader support and gesture handling
- __Consistent Behavior__: Works the same way across all Android apps
- __Automatic Timing__: Shows/hides appropriately without custom management
- __Professional Look__: Matches Android design guidelines

### Behavior:

1. __For touch devices__: Shows on long-press (not immediate touch)
2. __For devices with mouse/stylus__: Shows on hover
3. __Auto-dismiss__: Hides when user touches elsewhere or after timeout
4. __Positioning__: Automatically positions near the anchor view
